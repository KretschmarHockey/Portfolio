Named Entity Recognition

For Text and Vision Intelligence paper, we created a named entity recognition script. This reads TXT files in a folder and then saves the output as each file in different two folders. The first folder has a list of all the entities by category. The second folder has a list of all the entities with the category tag next to it.

This was achieved using the NLTK package. Once the files are read in, all the words are tokenized and then tagged on the words part of speech. The nouns are then chunked into noun phrases which are also tagged by entity recognition. With the test data the accuracy was 68.33%

Spam Detection

For Data Mining and Knowledge Engineering paper, we tested spam detection classifiers. This was done using the enron email corpus dataset. We needed lots of different options for training and testing our models.

We had two options for spliting the training and testing data. Either randomly splitting the emails into training and testing data, or using certain folders as training and testing data. We then had three different models to be trained. Naive Bayes, Neural Network or a Logistic Regression model.

The output of the program prints out the confusion matrix, classification report and accuracy of each model. Our most accurate model was the Neural Network when we randomly split the training and test data with 98.25% accuracy. Our worst performing model was the Neural Network when we chose specific folders for the training and testing data with 57.10% accuracy.

This was achieved using the NLTK package for pre-processing the unstructured data. We also used the sklearn package for vectorizing the words, randomly splitting the training and testing data, and using and analyzing the models.
